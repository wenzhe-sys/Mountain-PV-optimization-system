# 智能山地光伏电站设计优化平台

## 项目简介

智能山地光伏电站设计优化平台是一个基于React和TypeScript开发的现代化Web应用，旨在为山地光伏电站的设计和优化提供智能化解决方案。系统集成了多种先进算法，包括遗传算法、模拟退火、粒子群优化等，实现了从地形分析、面板布局、设备选址到电缆路由的全流程智能优化。

## 核心功能

### 1. 山地地形可视化
- 基于Perlin噪声的地形生成
- 坡度和坡向分析
- 太阳辐射计算
- 3D地形可视化
- 可建设区域识别

### 2. 光伏面板布局规划
- 遗传算法优化面板布局
- 模拟退火算法
- 粒子群优化算法
- 面板切割优化
- 分区管理

### 3. 电气设备选址
- 智能设备选址算法
- 负载平衡计算
- 电缆长度优化
- 多种设备类型支持

### 4. 电缆路由规划
- 智能路径规划
- 共沟敷设优化
- 成本分析
- 多种算法支持

### 5. 成本效益分析
- 建设成本分析
- 全生命周期成本
- 投资回报分析
- 优化效果对比

### 6. 生态影响评估
- 碳减排计算
- 植被保护分析
- 水土保持评估
- 环境影响分析

### 7. 运维监控中心
- 实时数据监控
- 设备状态监测
- 告警管理
- 环境参数监测

## 技术架构

### 前端技术栈
- React 18 + TypeScript
- Tailwind CSS
- Framer Motion (动画效果)
- Three.js + @react-three/fiber (3D地形)
- Recharts (数据可视化)
- Lucide React (图标库)

### 后端技术 (预留)
- Node.js + Express
- MongoDB
- Python (算法计算)

## 快速开始

### 环境要求
- Node.js 18+
- npm 9+

### 安装步骤

1. 克隆项目
```bash
git clone <项目地址>
cd <项目目录>
```

2. 安装依赖
```bash
npm install
```

3. 启动开发服务器
```bash
npm run dev
```

4. 构建生产版本
```bash
npm run build
```

## 使用指南

### 1. 系统登录
- 默认账号: admin@example.com
- 默认密码: password123

### 2. 地形分析
1. 进入"山地地形"模块
2. 选择可视化模式（海拔、坡度、坡向、太阳辐射）
3. 使用鼠标拖动和滚轮缩放查看地形
4. 点击"导出"按钮保存地形数据
5. 点击"导入"按钮加载地形数据

### 3. 面板布局优化
1. 进入"面板布局"模块
2. 点击"智能优化"按钮
3. 选择优化算法（遗传算法、模拟退火、粒子群优化）
4. 点击"开始优化"按钮
5. 查看优化结果和建议

### 4. 设备选址
1. 进入"电气设备"模块
2. 点击"智能选址"按钮
3. 设置优化目标和设备类型
4. 点击"开始优化"按钮
5. 查看设备布局结果

### 5. 电缆路由
1. 进入"电缆路由"模块
2. 点击"智能路由优化"按钮
3. 选择优化算法和电缆类型
4. 点击"开始优化"按钮
5. 查看优化后的路由方案

### 6. 成本分析
1. 进入"成本分析"模块
2. 查看建设成本、全生命周期成本和投资回报分析
3. 比较不同方案的成本效益

### 7. 生态影响
1. 进入"生态影响"模块
2. 查看碳减排、植被保护和水土保持分析
3. 了解项目的环境影响

### 8. 运维监控
1. 进入"运维监控"模块
2. 查看实时发电数据和设备状态
3. 处理系统告警
4. 监测环境参数

## 算法说明

### 1. 遗传算法
- 用于面板布局和设备选址优化
- 通过模拟自然选择和遗传变异寻找最优解
- 支持多目标优化

### 2. 模拟退火算法
- 用于局部搜索和精细优化
- 能有效避免局部最优解
- 收敛速度快

### 3. 粒子群优化
- 用于全局优化
- 并行搜索能力强
- 适合多目标优化问题

### 4. Benders分解
- 用于面板切割优化
- 能有效处理混合整数规划问题
- 计算效率高

## 系统配置

### 环境变量
在项目根目录创建 `.env` 文件：

```
VITE_APP_TITLE=智能山地光伏电站设计优化平台
VITE_APP_VERSION=2.0.1
VITE_API_URL=/api
```

### 部署配置
1. 构建生产版本：`npm run build`
2. 将 `dist` 目录部署到Web服务器
3. 配置反向代理（如需后端API）

## 开发指南

### 代码风格
- 使用 TypeScript 类型定义
- 遵循 React 最佳实践
- 使用 Tailwind CSS 进行样式设计
- 组件命名采用 PascalCase
- 函数命名采用 camelCase

### 提交规范
- feat: 新增功能
- fix: 修复 bug
- refactor: 代码重构
- docs: 文档更新
- style: 样式修改
- test: 测试代码
- chore: 构建过程或辅助工具的变动
